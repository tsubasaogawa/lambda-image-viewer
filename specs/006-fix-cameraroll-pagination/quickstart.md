# Quickstart: カメラロールのページネーション修正

## 1. 概要

このドキュメントは、実装されたページネーション修正機能の動作確認を迅速に行うための手順を記述します。

## 2. 前提条件

- AWS SAM CLI または Serverless Framework がインストールされていること。
- AWS認証情報が設定済みであること。
- プロジェクトの依存関係がインストール済みであること (`go mod tidy`, `npm install`)。

## 3. ローカルでの動作確認手順

1.  **Serverless Frameworkでローカルサーバーを起動**:
    ```bash
    # src/viewer ディレクトリに移動
    cd src/viewer

    # .env.tmpl を .env にコピーし、必要な環境変数を設定
    # (TABLE, REGIONなど)
    cp .env.tmpl .env
    # ... .env ファイルを編集 ...

    # ローカルサーバーを起動
    sls offline start
    ```

2.  **ブラウザで確認**:
    - Webブラウザを開き、ローカルサーバーのエンドポイント（通常は `http://localhost:3000`）にアクセスします。
    - カメラロールの最初のページが表示されることを確認します。

3.  **ページネーションの動作確認**:
    - ページ下部にある「次へ」ボタンをクリックします。
    - 表示される画像が、前のページよりも撮影日時が古いものだけになっていることを確認します。
    - 複数回「次へ」ボタンをクリックし、常に時系列が正しく保たれることを確認します。

4.  **最終ページの確認**:
    - 最後のページまで進みます。
    - 「次へ」ボタンが表示されなくなる、またはクリックできなくなることを確認します。

## 4. デプロイ

修正がローカルで正しく動作することを確認した後、以下のコマンドでAWSにデプロイできます。

```bash
# src/viewer ディレクトリで実行
sls deploy --stage your_stage
```
